## Crate Responsibilities

### ganesha-core

The heart of the system. Contains:

- **Planner**: Converts natural language tasks to execution plans
- **Executor**: Runs plans (shell commands, MCP tools, etc)
- **Verifier**: Wiggum loop - checks results, retries if needed
- **Memory**: Session persistence, learned patterns, goals
- **Rollback**: File snapshots, command history, undo
- **Sandbox**: Container/VM isolated execution
- **Mini-Me**: Subagent spawning and coordination

```rust
// Core engine entry point
pub struct GaneshaEngine {
    planner: Planner,
    executor: Executor,
    verifier: Verifier,
    memory: Memory,
    rollback: RollbackManager,
    minime: MiniMePool,
}
```

### ganesha-providers

LLM abstraction layer:

- Unified trait for all providers
- Auto-detection of available providers
- Model quality tier system
- Rate limiting and retry logic
- Streaming support

```rust
pub trait LlmProvider: Send + Sync {
    async fn chat(&self, messages: &[Message]) -> Result<Response>;
    async fn stream(&self, messages: &[Message]) -> Result<impl Stream<Item = String>>;
    fn model_tier(&self) -> ModelTier;
}
```

### ganesha-mcp

MCP server management:

- Hot loading/unloading
- Server registry (installed, available)
- Protocol implementation
- Credential management
- Built-in servers (search, fetch, shell)

```rust
pub struct McpManager {
    registry: ServerRegistry,
    loaded: HashMap<String, RunningServer>,
    config: McpConfig,
}
```

### ganesha-vision

Visual computer use:

- Cross-platform screen capture
- VLA model integration
- Mouse/keyboard control
- App whitelist/blacklist
- Safety systems

```rust
pub struct VisionController {
    capturer: ScreenCapturer,
    model: Box<dyn VlaModel>,
    input: InputController,
    safety: SafetyChecker,
}
```

### ganesha-voice

Voice I/O:

- Audio capture (microphone)
- Audio playback (speakers)
- OpenAI Realtime API
- Local STT/TTS fallback
- Personality system
- Wake word detection

```rust
pub struct VoiceController {
    input: AudioInput,
    output: AudioOutput,
    stt: Box<dyn SpeechToText>,
    tts: Box<dyn TextToSpeech>,
    personality: Personality,
}
```

### ganesha-cli

Terminal interface:

- Clap-based argument parsing
- REPL mode
- TUI mode (Ratatui)
- Output formatting
- Completions generation

### ganesha-desktop

Desktop companion app:

- Tauri 2.0 shell
- System tray integration
- Push-to-talk UI
- Glass bubble design
- Screen border overlay
- Provider discovery
